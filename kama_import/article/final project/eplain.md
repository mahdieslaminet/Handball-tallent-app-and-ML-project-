# پروژه دقیقاً چی کار می‌کنه؟
این پروژه یه سیستم سه‌مرحله‌ای برای هندباله که دو تا کار اصلی انجام می‌ده:  
1.	Talent Scoring از روی فرم متنی (Text)  
•	کاربر سن/قد/وزن/ساعت تمرین/سابقه و چندتا امتیاز ۱ تا ۱۰ (سرعت، چابکی، استقامت…) می‌ده  
•	سیستم می‌گه این آدم به کدوم پست بیشتر می‌خوره:  
Goalkeeper / Wing / Pivot / Backcourt  
2.	Pose Detection از روی عکس/ویدیو (Pose)  
•	با MediaPipe (مدل آماده) اسکلت بدن رو پیدا می‌کنه  
•	چند ویژگی ساده درمیاره مثل زاویه آرنج/زانو و visibility  
•	بعد دوباره یه پیشنهاد پست می‌ده (دمو)  
3.	پیش‌بینی نتیجه مسابقه با Elo  
•	از Elo تیم‌ها استفاده می‌کنه و احتمال برد میزبان / مساوی / برد میهمان رو می‌ده

# چه «مدلی» استفاده شده؟
اینجا نکته مهمه: این پروژه ML کلاسیکِ train شده نداره. بیشتر یه Prototype / Baselines هست.  

## 1) مدل مرحله Text
•	Rule-based scoring (یعنی قانون/فرمول دستی)  
•	برای هر پست یه امتیاز خام می‌سازه  
•	بعد با Softmax اون امتیازها رو تبدیل می‌کنه به درصدهای ۰ تا ۱۰۰  
پس “مدل” این بخش:  
✅ Heuristic Rule-based + Softmax normalization  

## 2) مدل مرحله Pose
•	خودِ Pose Estimation با MediaPipe Pose انجام می‌شه (مدل از قبل آموزش‌دیده)  
•	شما آموزش ندادید؛ فقط استفاده کردید + از خروجی‌اش ویژگی ساختید  
پس “مدل” این بخش:  
✅ Pretrained Pose Estimation (MediaPipe) + Rule-based scoring  

## 3) مدل مرحله Match Prediction
•	این ML نیست؛ یه مدل آماری/رتبه‌بندی معروفه: Elo Rating  
•	خروجی هم احتمالاته  
پس “مدل” این بخش:  
✅ Elo probabilistic model + home advantage + draw heuristic

# چه کلاسیفیکیشنی داریم؟
اینجا دو جور «کلاسیفیکیشن» می‌تونی بگی:  

## A) Multi-class classification (۴ کلاسه)
برای پیشنهاد پست:  
•	کلاس‌ها: Goalkeeper / Wing / Pivot / Backcourt  
•	روش تصمیم: بیشترین امتیاز (argmax) بعد از softmax  
✅ این از نظر مفهومی یک طبقه‌بندی چندکلاسه است، ولی Rule-based انجام شده نه ML supervised.  

## B) 3-class outcome prediction
برای بازی:  
•	کلاس‌ها: برد میزبان / مساوی / برد میهمان  
✅ این هم “کلاس‌بندی نتیجه بازی” هست ولی با Elo.

# سوال‌هایی که استاد ML ممکنه بپرسه (و جواب آماده)
این‌ها خیلی محتمل‌اند:  

## درباره «مدل و یادگیری ماشین»
1.	اینجا دقیقاً ML دارید یا نه؟  
جواب: Pose بخش AI آماده (MediaPipe) هست، اما تصمیم‌گیری نقش و Talent فعلاً baseline rule-basedه. نسخه بعدی می‌تونه با دیتای واقعی supervised ML بشه.  
2.	چرا مدل یادگیری‌ماشین train نکردید؟  
جواب: چون دیتای برچسب‌دار نقش/پرفورمنس نداشتیم؛ هدف پروژه پیاده‌سازی pipeline و demo قابل ارائه بوده. (و این baseline برای مقایسه خوبه)  
3.	اگر دیتاست داشتید، چه مدلی train می‌کردید؟  
جواب پیشنهادی:  
•	برای Text: Logistic Regression / RandomForest / XGBoost  
•	برای Pose features: SVM یا XGBoost  
•	خروجی: ۴کلاسه (role classification)  
4.	Featureها چی هستن؟  
•	Text: سن، BMI، ساعت تمرین، سابقه، سرعت/چابکی/…  
•	Pose: زاویه آرنج و زانو + visibility  
5.	Label (برچسب) چی می‌شه؟  
برچسب می‌تونه “پست واقعی بازیکن” یا “پست پیشنهادی مربی” باشه.  
6.	متریک ارزیابی چی می‌ذارید؟  
Accuracy، Macro-F1 (چون کلاس‌ها ممکنه نامتوازن باشن)، Confusion Matrix  
7.	Train/Test Split یا Cross-validation چی؟  
۸۰/۲۰ یا ۵-Fold CV برای داده کم.  
8.	Overfitting چیه و چطور کنترلش می‌کنید؟  
مدل پیچیده روی داده کم حفظ می‌کنه؛ با CV، regularization، محدود کردن depth در treeها، early stopping…  
9.	Data leakage یعنی چی؟  
ویژگی‌هایی که از آینده یا از label لو بره. (مثلاً آماری که بعد از انتخاب پست ثبت شده)  
10.	نرمال‌سازی/اسکیلینگ لازم دارید؟  
برای مدل‌هایی مثل SVM/LogReg آره. برای Tree-based کمتر.  

## درباره Softmax و امتیازدهی
11.	Softmax چرا استفاده شده؟  
برای اینکه امتیازهای خام به شکل درصدهای قابل‌فهم و قابل مقایسه تبدیل بشن و جمع‌شون ۱۰۰ بشه.  
12.	این درصدها “احتمال واقعی” هستن؟  
نه دقیقاً؛ بیشتر “امتیاز نرمال‌شده” هستن. برای احتمال واقعی باید مدل probabilistic train شده داشته باشیم.  

## درباره Pose و MediaPipe
13.	MediaPipe دقیقاً چی می‌ده؟  
مختصات landmarkهای بدن (keypoints) + visibility هر نقطه.  
14.	زاویه‌ها چطور حساب می‌شن؟  
با سه نقطه (مثلاً شانه-آرنج-مچ) و فرمول زاویه بین دو بردار (dot product).  
15.	محدودیت Pose چیه؟  
نور بد، پوشش بدن، زاویه دوربین، خارج شدن بخشی از بدن از کادر، ویدیو کم‌کیفیت.  

## درباره Elo
16.	Elo چیه؟ چرا استفاده کردید؟  
مدل رتبه‌بندی سبک و معروف؛ فقط به نتایج برد/باخت نیاز داره و سریع جواب می‌ده.  
17.	home advantage چی هست؟  
یه بایاس ثابت به تیم میزبان برای واقع‌گراتر شدن احتمال‌ها.  
18.	مساوی رو چطور مدل کردید؟  
یک heuristic: وقتی اختلاف Elo کم باشه احتمال مساوی بیشتر می‌شه.  
19.	چطور Elo رو validate می‌کنید؟  
Backtesting: بازی‌های گذشته رو یکی‌یکی شبیه‌سازی کنیم و logloss / accuracy نتایج احتمالی رو بسنجیم.
